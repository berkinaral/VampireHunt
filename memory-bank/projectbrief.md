# Project Brief: Vampire Hunt Mobile Game

## Project Overview
A mobile social deduction game where friends nearby can join the same room using a shared online code. Players are randomly assigned roles (Vampire or Villager) and the app acts as an automated moderator to control game flow, similar to "Town of Salem".

## Core Requirements

### Functional Requirements
1. **Room Management**
   - Create rooms with unique shareable codes
   - Join rooms using codes
   - Support multiple players in same room
   - Host/creator controls to start game

2. **Role Assignment**
   - Random role distribution (Vampires vs Villagers)
   - Secret role reveals to individual players
   - Maintain role secrecy from other players

3. **Game Flow Management**
   - Automated moderator system
   - Day/Night cycle management
   - Timer controls for discussion phases
   - Voting round management
   - Game state tracking

4. **Voting System**
   - Night voting for vampires (select victim)
   - Day voting for all players (eliminate suspect)
   - Vote tallying and result announcements
   - Anonymous voting options

### Non-Functional Requirements
- Simple and intuitive UI
- Fun and engaging visual design
- Social-focused experience
- Real-time synchronization between players
- Mobile-optimized interface
- Quick game sessions (10-20 minutes)

## Target Audience
- Groups of friends (4-12 players)
- Social gatherings
- Party game enthusiasts
- Ages 13+

## Success Criteria
- Players can easily create/join rooms
- Game flow is smooth and automated
- Roles remain secret until revealed
- Voting is fair and transparent
- UI is clear and engaging
- Minimal technical issues during gameplay

## Constraints
- Mobile-first design (iOS and Android)
- Real-time multiplayer functionality required
- Must work on local networks and internet
- Simple enough for first-time players
- Scalable to handle multiple concurrent games
